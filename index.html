AQS 85–100
High confidence
Address verified across multiple dimensions. Strong pincode match, locality confirmed, specific enough for delivery. Typically used for auto-approval in lending flows.
AQS 50–84
Medium confidence
Partial verification. Some dimensions scored well but others flagged — locality unconfirmed, low specificity, or geocode mismatch. Most NBFCs trigger additional verification here.
AQS 0–49
Low confidence
Significant quality issues. Flags may include gibberish patterns, locality not in pincode, city-state mismatch. Evidence flags explain exactly why. Most NBFCs require manual review or field verification.

Evidence flags tell you why

The score tells you how much to trust. The flags tell you what's wrong.

[flag]

CITY_STATE_MISMATCH

User-provided city or state doesn't match pincode truth. The pincode says Karnataka but the applicant wrote Maharashtra.

[flag]

LOCALITY_NOT_IN_PINCODE

The locality claimed doesn't exist in this pincode's known localities. Could be a typo, could be fabricated.

[flag]

GIBBERISH_DETECTED

Address text shows adversarial patterns — consonant clusters, repeated characters, keyboard-mash sequences.

[flag]

LOW_GRANULARITY

No house number, flat number, or street name found. Address lacks the specificity needed for physical delivery or field verification.

[flag]

NO_GEOCODE_HIT

Address couldn't be geocoded. This is a soft signal — valid new colonies may not be in any database yet.

[flag]

FRAUD_SUSPICIOUS_ATTEMPTS

Multiple low-score attempts from this device/session within the configured window. Pattern suggests address cycling.

Lookup the pincode

Validates the pincode and returns a session token, state, district, and known localities. The session token is required for scoring.

curl
Python
Node.js
curl -X POST https://api.aqs.in/v1/pincode/lookup \
  -H "Authorization: Bearer aqs_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"pincode": "560001"}'

# Response includes session_token, state, district, localities
import requests

lookup = requests.post(
    "https://api.aqs.in/v1/pincode/lookup",
    headers={"Authorization": "Bearer aqs_YOUR_KEY"},
    json={"pincode": "560001"},
)
session_token = lookup.json()["session_token"]
# Also returns: state, district, localities
const lookup = await fetch("https://api.aqs.in/v1/pincode/lookup", {

    "Authorization": "Bearer aqs_YOUR_KEY",

  },
  body: JSON.stringify({ pincode: "560001" }),
});
const { session_token } = await lookup.json();

Score the address

Use the score in your LOS

The score and flags feed into your existing decisioning logic. AQS scores, you decide.

Node.js
score = result["aqs_score"]
flags = result["flags"]

# Your decisioning logic — AQS doesn't prescribe actions
if score >= 85 and not flags:
    approve_loan(application_id)

elif score >= 50:
    request_additional_kyc(application_id, flags=flags)

elif "GIBBERISH_DETECTED" in flags:
    reject_application(application_id, reason="Address quality insufficient")

else:
    trigger_field_verification(application_id)
